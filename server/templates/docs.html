Documentation

An overview of the Scholarly API. The page describes how to initialise the Scholarly API on JavaScript and Python. The API aims to making Scholarly products easy and accesible to those with a license.

Functions #

Here is an overview of all the functions provided by Schoarly. All JavaScript versions of the functions are asynchronous requiring an await statement. These are not finished, so pay attention to the following:

  • coach(prompt) — Make a chat request with Scholarly's custom language model, Coach. Prompt is an argument for the question. [INCOMPLETE]
  • markImage(img) — Mark and provide feedback to handwritten work via images. Takes in an image file of PNG/JPG. [INCOMPLETE]
  • markFile(pdf) — Mark and provide feedback to digital work via pdf. Takes in a PDF file argument. [INCOMPLETE]
  • getKey() — Recieve OpenAI keys for custom services not yet provided via API. No arguments. [AVAILABLE]

Javascript Module #

This module is both available as a local file and via CDN.

Download JS Module

Firstly, check your JS file is linked to as type = "module" if you are operating through HTML.

JavaScript example using CDN 
import { getKey } from "https://api.scholarly.repl.co/cdn-js";
                              
var openai_key = await getKey();
if (openai_key == undefined) {
  //Handle no key found logic here
}

Pleast note, the JS file has other functions than getKey. In this example, it is used to showcase how to import the file and what to expect. For example, adding the coach() function to recieve messages from the custom language model is also available. It is up to you what modules you add to import.

Where To Next #

Once you have downloaded and setup the modules of choice, the possibilites are endless. As more functions appear, devs will be able to build a Scholarly app on any platform, as all backend databases, algorithms, etc. will be accesible via API. This allows cross-platform with the luxury of no major back-end hastles.

All code is available for devs to see, and for now, no in-depth documentation on the functions needs to be released. As well as this, many functions aren't ready yet, and their status will be updated.
